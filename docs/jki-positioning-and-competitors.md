# JKI Positioning & Competitor Analysis

## 🎯 The Soul of JKI
JKI (Just Keep Identity) is the spiritual successor to **WinAuth**, rebuilt for the modern macOS/Linux terminal. It exists because identity management on desktop OSs has become bloated, slow, and overly dependent on GUI interactions.

## ⚖️ Core Manifesto
> **Getting auth codes should never require a mouse.**

For a CLI power user, the friction of taking hands off the home row to click a search box in a GUI app is a productivity bug. JKI aims to eliminate this friction entirely.

## ⚔️ Competitor Analysis

| Feature | WinAuth (Legacy) | Step Two / Raivo (GUI) | Bitwarden / 1Password CLI | **JKI (CLI)** |
| :--- | :--- | :--- | :--- | :--- |
| **Interface** | Lightweight GUI | Polished GUI | Heavy CLI Wrapper | **Pure TTY / Rust** |
| **Input Method** | Keyboard/Mouse | **Mouse Required** | CLI Commands | **Micro-Roll (Home row)** |
| **Latency** | Near zero | High (UI interaction) | High (Sync/Unlock wait) | **Zero (Local Agent)** |
| **Trust Model** | Local File | Proprietary Cloud | Corporate SaaS | **Git-Synced (SSoT)** |
| **Philosophy** | "Keep it simple" | "Make it pretty" | "Everything in one box" | **"Stay in the flow"** |

## 🚀 The Blue Ocean Strategy
JKI does not compete with general-purpose password managers. It targets a specific niche: **Engineers and Power Users** who:
1. Live in the terminal.
2. Value physical integrity (Secrets pinned to hardware).
3. Demand the speed of a shell alias for their auth tokens.
4. Want to own their infrastructure via Git.

## 📣 Messaging Guidelines
- **NEVER** use "Stop reaching for your phone" (Too generic/marketing-heavy).
- **NEVER** compare speed to a shell alias directly (Inaccurate/Misleading).
- **ALWAYS** focus on the elimination of GUI friction and "Home row" ergonomics.
- **ALWAYS** be honest about the backstory (WinAuth nostalgia + macOS frustration).
