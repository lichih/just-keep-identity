# Mission Report: Vault State & Transition Implementation

## 1. 實作摘要 (Implementation Summary)

本任務完成了「金庫狀態感知 (Vault State Awareness)」機制與相關搜尋邏輯的升級。

### 核心變動內容：
- **jki-core (`paths.rs`, `lib.rs`)**:
    - 新增 `decrypted_secrets_path()` 支援 `vault.secrets.json` 定義。
    - 驗證 `fuzzy_match` 確保字元消耗順序嚴格。
    - 驗證 `search_accounts` 實作「欄位隔離」：單一 Pattern 僅在 `Issuer` 或 `Account Name` 之一匹配。
- **jki (`main.rs`)**:
    - 重構 `run()` 流程，導入搜尋優先序：
        1. **Plaintext Path**: 直接讀取 `vault.secrets.json` (最快)。
        2. **Agent Path**: IPC 請求 `jki-agent`。
        3. **Static Key Path**: 使用 `master.key`。
        4. **Interactive Path**: 提示輸入。
- **jkim (`main.rs`)**:
    - 新增 `decrypt` 指令：將 `.age` 轉為 `.json` 明文。
    - 新增 `encrypt` 指令：將 `.json` 轉為 `.age` 並刪除明文。
    - 重構 `import-winauth`:
        - 支援 `Hybrid` 狀態感知：偵測到 `master.key` 時，匯入後自動執行加密封裝。
        - 支援 `-y, --yes` 旗標跳過衝突確認。
        - 修正並新增相關單元測試。
