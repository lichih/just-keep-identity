# Mission: Just Keep Identity (jki) - Phase 4 (Agency & Key Caching)

## 1. Project Context
JKI has transitioned from a standalone CLI to a managed vault suite with Git automation and a basic background agent foundation.
- **Standalone Success**: `jki` and `jkim` are fully functional with manual/file-based master key unlock.
- **Management Pro**: `jkim` supports atomic key rotation and safe metadata editing.
- **Testing Mastery**: `Interactor` pattern allows 100% automated testing of interactive auth flows.
- **IPC Ready**: `jki-agent` exists with a verified JSON-over-socket protocol.

## 2. Objectives for this Session
### Objective A: Intelligent Agency (`jki-agent` Core)
- Implement `Request::Unlock` to allow the agent to verify a master key and load the secrets vault into memory.
- Implement `Request::GetOTP` logic within the agent, returning calculated TOTP tokens directly via IPC.
- Update `jki` to automatically prefer the Agent for OTP generation if available, achieving zero-prompt usage.

### Objective B: Key Caching & Security
- Implement memory-only session caching for the Master Key (or decrypted secrets map).
- Add a configurable Session TTL (e.g., 1 hour) after which the agent wipes sensitive data from memory.
- Research and implement `Zeroize` for active memory clearing of keys.

### Objective C: System Integration (Keychain/DPAPI)
- Implement platform-specific backends for secure Master Key storage:
    - **macOS**: `security` framework (Keychain Services).
    - **Windows**: `dpapi` (Data Protection API).
- Allow the agent to automatically "auto-unlock" on startup if a key exists in the system keychain.

### Objective D: Service Refinement
- Implement an auto-start mechanism: if `jki` cannot connect to the socket, it should attempt to spawn `jki-agent` in the background.
- Improve IPC error handling: define more granular `Response::Error` types.

## 4. Key Constraints
- **Fail Fast**: Maintain the "no retry" philosophy for password entry.
- **Security First**: Secrets must NEVER be written to disk in plaintext by the agent.
- **Portability**: Ensure IPC works across Unix Domain Sockets and Windows Named Pipes (abstracted by `interprocess`).
